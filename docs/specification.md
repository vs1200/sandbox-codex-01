# tetris-ren-ren 仕様書

## 1. 概要

テトリスの **4列REN（連鎖）** を練習するためのWebアプリケーション。
プレイヤーは実際にミノを操作するのではなく、提示された有効な配置選択肢の中から正しいものを選んでRENを繋いでいく**意思決定トレーニングツール**である。

### 1.1 コンセプト

- 10×20のテトリスボードの中央4列のみを使用
- 左3列・右3列は常にブロックで埋まっている（「中開け」配置）
- ミノの配置先をボタン選択する形式で、操作スキルではなく**配置判断力**を鍛える
- 参考実装: https://shiwehi.com/tetris/game/i4lr.php

## 3. ゲームロジック

### 3.1 ボード構造

- **グリッドサイズ**: 10列 × 20行
- **プレイエリア**: 列3〜6（0始まり）の中央4列
- **壁エリア**: 列0〜2、列7〜9 は灰色ブロックで常時充填
- **現在ミノ表示**: ボード上部（row 0〜1）に現在のミノの形状を色付きで表示する
- セル番号は `row * 10 + col` で表現（row: 0〜19、col: 0〜9）

### 3.2 ミノ（テトリミノ）

標準7種のテトリミノを使用する。

| ミノ | 色名 | カラーコード |
|------|-------|-------------|
| I | シアン | `#00b7ff` |
| O | イエロー | `#ffff00` |
| J | ブルー | `#0011ff` |
| L | オレンジ | `#ff7b00` |
| S | グリーン | `#3cff00` |
| Z | レッド | `#ff0000` |
| T | パープル | `#8400ff` |

> **注記**: 上記のカラーコードは参考実装の値。ダークモードUIでの視認性を考慮し、最終的な色は調整する可能性がある。

### 3.3 7-bag ランダマイザー

- 7種のミノを1セットとし、シャッフルして順番に配る
- 1セット配り終わったら次のセットをシャッフルして追加
- キュー内のミノが7個未満になったら新しいセットを追加する

### 3.4 タネ（盤面パターン）システム

ゲームの盤面状態を2桁の数値「タネ」で抽象化して管理する。

- **形式**: `ntj * 10 + nti`（ntj: 十の位、nti: 一の位）
- タネはライン消し後にプレイエリア下部に残るブロックのパターンを表す

#### タネパターン一覧

| ntj | 説明 | nti の範囲 |
|-----|------|-----------|
| 1 | 1列に3段積み | 1〜4（各列に対応） |
| 2〜5 | 1セル（中段）+ 2セル（最下段） | 1〜6（2セルの組み合わせ） |
| 6 | 2セル（中段）+ 1セル（最下段） | 1〜8（各パターンに対応） |
| 7 | 最下段に3セル | 1〜4（空き1列の位置に対応） |

#### 初期タネ

ゲーム開始時、以下の値からランダムに1つ選択される:

```
[21, 23, 53, 56, 61, 66, 71, 74]
```

### 3.5 配置データベース（choicedb）

有効な配置パターンを定義するデータベース。

- **構造**: `[ミノ種, 現在のタネ, 次のタネ]` の三つ組の配列
- ミノ種（`"i"`, `"o"`, `"j"`, `"l"`, `"s"`, `"z"`, `"t"`）と現在のタネから、配置後に遷移可能な次のタネを検索する
- 参考実装のデータをそのまま移植する

#### 検索ロジック

```
入力: currentMino, currentTane
出力: nextTane の配列（有効な配置先の候補一覧）
```

- 現在のミノ + 現在のタネで choicedb をフィルタリングし、遷移可能な次のタネ一覧を取得
- HOLDミノについても同様に検索（HOLDミノが現在のミノと異なる場合のみ）

### 3.6 HOLD システム

- ミノを1つ保持できる
- ゲーム開始時のHOLDは空（未有効状態）
- `holdActivated` フラグでHOLDの有効/無効を管理する

#### HOLD 未有効時（holdActivated = false）

- 配置候補と並んで **HOLDボタン** が表示される
- HOLDボタンをクリックすると、現在のミノをHOLDに格納し、キューの次のミノに進む
- この操作で `holdActivated = true` になる

#### HOLD 有効時（holdActivated = true）

- 毎ターン、現在ミノの配置候補に加えて **HOLDミノの配置候補** も自動的に表示される
  - ただし、HOLDミノと現在ミノが同じ種類の場合は候補が重複するため、HOLD候補は表示しない
- HOLDミノの配置候補を選択すると、HOLDミノで配置され、現在のミノがHOLDに自動的に入る（スワップ）

### 3.7 NEXT キュー

- 次に来るミノを **5個先** まで表示する
- 7-bag ランダマイザーからミノが供給される

### 3.8 REN カウント

- 連続でライン消しが成功した回数をカウント・表示する
- ゲーム開始時は `-1`（最初のライン消しで `0` になり、表示は `1 REN` から）

### 3.9 ゲームオーバー条件

- HOLD有効時：現在ミノの配置候補とHOLDミノの配置候補の**両方が空**の場合にゲームオーバー
- HOLD未有効時：現在ミノの配置候補が空でも、HOLDボタンでミノを格納して次のミノで続行できるため、即座にゲームオーバーにはならない

## 4. ゲームモード

### 4.1 無限モード（Infinite）

- できるだけ長くRENを繋ぎ続けるモード
- ゲームオーバー時にREN数を表示
- タイマーは表示しない

### 4.2 タイムアタック（TA）モード

- **25 REN** をどれだけ早く達成できるかを競うモード
- ゲーム中にタイマーを表示（ミリ秒精度）
- 25 REN達成時にタイマー停止、記録を表示
- 25 REN達成前にゲームオーバーになった場合は通常のゲームオーバー

## 5. ゲームフロー

### 5.1 開始フロー

```
モード選択 → ボードリセット → 初期タネ設定 → ミノキュー生成
→ 現在ミノ・NEXT表示 → 配置候補検索・表示 → （TAモード: タイマー開始）
```

### 5.2 プレイループ

```
配置候補をクリック
  ├── シャッフル選択 → ボードリセット＆再スタート
  ├── HOLD選択 → HOLDにミノ保存、holdActivated = true
  └── 配置選択
        ├── HOLDミノの配置 → HOLDミノと現在ミノを交換
        └── 現在ミノの配置 → そのまま
        → RENカウント増加
        → タネ更新
        → ミノキュー進行（shift）
        → キュー補充チェック
        → ボード・アニメーション・チョイス・NEXT・HOLD再描画
        → 次の配置候補検索・表示
            ├── 候補あり → ループ続行
            └── 候補なし → ゲームオーバー判定（→ 3.9 参照）
```

### 5.3 ゲームオーバー

- REN数を表示
- TAモードの場合: タイム記録も表示
- リスタートオプションを提供

### 5.4 シャッフル

- ゲーム開始直後（最初のミノ配置前）のみ使用可能
- ミノキュー・タネをすべてリセットして再生成
- TAモードではタイマーもリセット

## 7. 状態管理（Zustand）

### 7.1 ストア構造

```typescript
interface GameState {
  // ゲーム状態
  gameStatus: 'idle' | 'playing' | 'gameover';
  mode: 'infinite' | 'timeAttack';

  // ミノ管理
  minoQueue: MinoType[];       // ミノキュー（14個程度を維持）
  holdMino: MinoType | null;   // HOLDミノ
  holdActivated: boolean;      // HOLDが有効化されたか（初回格納済みか）

  // 盤面
  tane: number;                // 現在のタネ値
  ntj: number;                 // タネの十の位（派生値）
  nti: number;                 // タネの一の位（派生値）

  // スコア
  ren: number;                 // RENカウント

  // 配置候補
  nextChoices: number[];       // 現在ミノの配置候補
  holdChoices: number[];       // HOLDミノの配置候補

  // アニメーション（ボード表示用の最小コンテキスト）
  animation: AnimationState | null;

  // タイマー（TAモード）
  startTime: number | null;
  elapsedTime: number;
  timeResult: string;

  // アクション
  startGame: (mode: GameMode) => void;
  selectChoice: (choiceId: string) => void;
  shuffle: () => void;
  resetGame: () => void;
}
```

### 7.2 型定義

```typescript
type MinoType = 'i' | 'o' | 'j' | 'l' | 's' | 'z' | 't';
type GameMode = 'infinite' | 'timeAttack';
type GameStatus = 'idle' | 'playing' | 'gameover';
type AnimationPhase = 'placing' | 'clearing';

interface AnimationState {
  phase: AnimationPhase;
  prevTane: number;          // アニメーション開始時点のタネ（ボード表示用）
  placedMino: MinoType;      // 配置されたミノの種類
  targetCells: number[];     // ミノが配置されるセル位置
}

interface ChoiceEntry {
  mino: MinoType;
  currentTane: number;
  nextTane: number;
}
```

### 9.1 参考実装の方式

参考実装では各配置パターンを画像ファイル（`{ミノ種}{現タネ}{次タネ}.png`）で表示している。

### 9.2 本実装の方針

配置パターンを**プログラムで描画**する方式に変更する。

- タネパターンのデータから動的にプレビューを生成する

## 10. 配置アニメーション

配置選択を決定した際に、ミノがボード上に配置される過程を素早いアニメーションで表現する。

### 10.1 設計方針

ゲームロジックの状態とアニメーション表示を分離し、ノンブロッキングで実現する。

- ゲームロジック（`tane`, `minoQueue`, `nextChoices` 等）は配置決定時に**即座に更新**される
- 配置候補パネル、NEXT、HOLD、RENカウンターは最新のゲーム状態を即座に反映し、ユーザーはすぐに次の操作が可能
- ボード表示のみ、アニメーション用の最小コンテキスト（`animation` フィールド）を参照して表示を切り替える

### 10.2 アニメーションフロー

```
selectChoice() 実行
  │
  ├── ① ゲームロジック即時更新
  │     tane → 新タネ、minoQueue.shift()、nextChoices → 新候補、ren++
  │     → ChoicePanel, NEXT, HOLD, REN は即座に新状態で描画
  │
  └── ② animation フィールドを設定
        animation = { phase: 'placing', prevTane: 旧タネ, placedMino, targetCells }
        → BoardGrid は prevTane ベースでミノ落下アニメーションを描画
        → 250ms後: phase → 'clearing'（ライン消しエフェクト）
        → 200ms後: animation → null
        → BoardGrid は最新の tane で通常描画に復帰
```

### 10.3 アニメーションコンテキスト

Zustand ストアの `animation` フィールドに以下の最小限の情報を保持する（ゲーム状態のフルコピーではない）。

| フィールド | 用途 |
|------------|------|
| `phase` | 現在のアニメーション段階（`'placing'` / `'clearing'`） |
| `prevTane` | アニメーション中にボードに表示する旧タネ値（数値 1つ） |
| `placedMino` | 配置されたミノの種類（落下描画用） |
| `targetCells` | ミノが配置されるセル番号（落下先の位置） |

### 10.4 ボードの描画切り替え

| `animation` の状態 | BoardGrid の描画内容 |
|-------------------|---------------------------|
| `null`（通常時） | 現在の `tane` で盤面を描画 |
| `phase: 'placing'` | `prevTane` の盤面 + `placedMino` の落下アニメーション |
| `phase: 'clearing'` | 完成行のフラッシュ/フェードアウトエフェクト |

### 10.5 高速連打時の処理

- アニメーション再生中に次の配置が行われた場合、現在の `animation` を即座に `null` にリセットし、新しいアニメーションを開始する
- ボード表示は一瞬で最新状態にジャンプしてから新アニメーションを再生

### 10.6 タイミング

- **落下アニメーション**（`placing` フェーズ）: 200〜300ms
- **ライン消しエフェクト**（`clearing` フェーズ）: 150〜200ms
- **合計**: 400〜500ms 以内
